Exercise 4: String Assignment

What does this code log to the console?

Answer

This should log Bob Bob to the console, since name and saveName both point to the same String 'Bob'. The toUpperCase() method returns a new string representing the calling string converted to upper case, but neither variable is reassigned to this new string, so are still assigned to 'Bob'.

Discussion

If you were thinking that at least one or both should have been uppercase then you would have been wrong. Don't worry, you are not alone, especially if you come from other programming languages where strings are mutable. However, as seen in the result, in JavaScript strings are primitive and thus immutable.

Further Exploration

Taking another look at the code, you will notice that in line 3 the name variable calls a method, the String.prototype.toUpperCase() method. How is it possible that strings are primitive but still able to call methods?
