Exercise 5: Arithmetic Integer

In this program we're going to explore the basic arithmetic operations in JavaScript. The program below prompts the user for two positive integers and then logs the results of the following operations on those two numbers: addition, subtraction, product, quotient, remainder, and power. For this exercise, assume positive integers as inputs.

Here is our expected result after running the program.

Enter the first number: 23
Enter the second number: 17

23 + 17 = 40
23 - 17 = 6
23 * 17 = 391
23 / 17 = 1
23 % 17 = 6
23 ** 17 = 1.4105003956066297e+23
Did the results of the program match what you expected? Why or why not? If not can you fix it?

Answer

The results did not match the 'expected results' due to the implicit coercion which takes place in JavaScript when using operators. The input to the prompts are strings, the operators are then used on these strings. The results and types of coercion that take place depend on the combination of operators and the data type of the operands. For example, + when used with two strings concatenates the strings and performs no type coercion, so the result of '23' + '17' is a new string '2317', but the result of '23' - '17' is the integer 6 since when using the - operator with string operands, JavaScript coerces both of the string operands to integers.

In order to correct the program, we can explicitly coerce the string input to numbers, e.g. by using parseInt() prior to usng those numbers in our calculations.

Discussion

JavaScript does a lot of implicit coercion behind the scenes that can lead to unexpected results. The solution makes use of explicit primitive type coercions, via parseInt, so that we have better control of the outcome. The second argument for parseInt is a radix that specifies the number parsed as base 10.
